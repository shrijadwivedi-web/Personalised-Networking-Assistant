# 6. Project Testing

The repo includes tests for:

- authentication
- routes
- error handling
- event analysis
- fact checking
- feedback logging
- history logging
- scorer logic
- topic generation

- Correct HTTP responses
- Account isolation

- Safe fallback behavior when services are unavailable

- Unit tests for service logic

- Error path tests for runtime failures
